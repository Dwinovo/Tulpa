# 言出法随 Numen —— 尽管开口，剩下的交给我

**言出法随（Numen）** 为 Minecraft 加入一个由大语言模型（LLM）驱动的 AI 伙伴。你用自然语言——任意语言——和它对话，它就用自己在世界里的"身体"去完成多步骤的工作：寻路、挖矿、建造、战斗、合成、使用容器。它不是一套预设脚本，而是**模型负责思考、身体负责执行**：模型表达"意图"，伙伴自己想办法（怎么走、用什么工具、够不够得着）把事做完。

> 名字的由来：英文名 Numen 源自藏传佛教的"化身（sprul-pa）"，后被引申为"以专注的意念创造、并能独立行动的思维体"——这正是本模组的内核：**一个由你的心智（你电脑上的 LLM）创造、却能在世界中自主行动的存在**。中文名取**「言出法随」**——你说一句人话，它便把事做成，恰好就是这个模组的核心体验。

---

## 一句话理解

召唤一个伙伴，然后就**和它说话**。它的身体是一个真正的玩家实体，而驱动它的"大脑"运行在**你自己的电脑上、使用你自己的 API Key**。

例如，你只需要说：

> *"去挖一组铁矿回来"* · *"在我站的地方盖个小屋"* · *"跟着我，见到敌对生物就打"* · *"把这些矿石烧了，铁锭放进那个箱子"*

剩下的交给它。

---

## 核心特性

- **直接对它说话。** 自然语言请求会变成世界里真实的、多步骤的工作——没有脚本，没有一屏的开关菜单。

- **大脑跑在"你"的机器上。** 智能体循环（agent loop）从**主人的客户端**调用 LLM，用的是**主人自己的 API Key**。没有服务端 LLM，没有共享账单——每位玩家只为自己伙伴的"思考"付费。

- **身体是真正的玩家，而非自定义生物。** 伙伴是服务端的*假人（fake player）*，通过原版玩家代码与世界交互。因此它能和其他模组、红石、生物 AI、各类容器**天然兼容**，而不需要逐个功能去写适配。

- **用 Markdown 文件就能教它玩任何模组——无需适配、无需等待。** 因为伙伴是真实玩家、且使用的是*通用*工具（`transfer`、`interact_at` 能操作**任意** GUI 或方块，无论原版还是模组），让它学会一个新模组靠的不是兼容补丁，而是一个用大白话写的 `SKILL.md`。想让它运营机械动力的安山岩农场、或推进通用机械的科技线？把流程写下来，文件丢进去即可。**你不必等模组作者，也不必等我们。**（详见下方"技能系统"）

- **27 个工具。** 挖掘、建造、战斗（原版近战 + 弓箭）、把物品手动摆进合成格来合成、使用任意容器、扫描世界、定位结构/生物群系、管理物品栏，并用一份实时待办清单来规划。

- **自带你想用的模型。** 开箱支持 OpenAI、DeepSeek、Kimi / Moonshot、MiniMax、豆包 / 火山引擎、通义千问 / DashScope——任何 OpenAI 兼容接口都可通过自定义 Base URL 接入。

## 技能系统（Skill）—— 用 Markdown 教它玩任何东西

Numen 与世界交互的方式是**自然语言**，从头到尾都是；你**扩展**它的方式同样如此。

一个**技能**就是一个 `SKILL.md` 文件：YAML 前言（`name` + `description`）加上一段用大白话写的操作说明。没有代码，没有 Java，无需重新编译。

这是"假人"设计带来的红利。伙伴本就通过*通用*工具触碰世界——`transfer` 和 `interact_at` 能像人手一样操作**任意**容器、机器或方块，原版或模组皆可。所以教它一个新模组从来不是一个工程项目，而是一次"写说明书"。**你不必等模组作者加入支持，也不必等我们发补丁**——你把流程写下来，伙伴就能照做。

例如，一个教它玩机械动力的技能文件：

```
config/numen/skills/create_andesite_farm/SKILL.md
─────────────────────────────────────────────────
---
name: create_andesite_farm
description: 搭建并运行机械动力的安山岩生成器——组装动力链、对准料盘、囤积安山岩。
---
# Skill: create_andesite_farm

1. 合成水车与安山岩外壳（分别用 lookup_recipe 查配方）。
2. 把水车放在流水上方让它转起来；……
3. ……（剩下的步骤，用你自己的话写）
```

把这个文件夹丢进去，伙伴就会"玩"机械动力了——同样的招数也能教它通用机械、农夫乐事、应用能源，或你自己整合包的进度线，只要你能把流程描述清楚。

**模型如何使用技能——渐进式披露，让上下文保持精简：**

- **每一轮对话**，模型只看到一份*目录*：每个技能的 `name` + `description`，仅此而已。哪怕装了几十个技能，开销也几乎为零。
- **当某个任务匹配上**某条描述时，模型才调用 `load_skill` 工具，把**技能正文完整地载入对话**——按需提供、用到才花 token。

**技能存放位置：** `config/numen/skills/<名称>/SKILL.md`。Numen 内置了一套技能——战斗基础、容器/合成/烧炼操作手册，以及一条完整的"下界 → 烈焰棒 → 末影珍珠 → 要塞 → 末影龙"速通链条——首次启动时会被释放到该文件夹。此后**这个文件夹就归你了**：可以修改、删除、写自己的。它与更广泛的 Agent Skill 生态采用同一种 `SKILL.md` 格式，所以在别处写好的技能多半能直接拿来用。

## 工作原理

```
   你（主人的客户端）                              服务端
   ─────────────────                             ──────────────────────────
   聊天框 ─▶ 智能体循环 ──▶ LLM 接口
                  │ （你的 Key、你的模型）
                  │ 工具调用
                  ▼
            ExecuteToolPayload ───────────▶  假人身上的任务队列
                  ▲                              │ 逐 tick 分发器
                  │  结果                        ▼
            TaskResultPayload ◀──────────   寻路 · 挖矿 · 战斗 ·
                                            合成 · 使用容器 ……
```

1. 你输入一句话；**客户端侧的智能体循环**把对话与工具定义发给你选定的 LLM，并流式接收回复。
2. 只读类工具（状态、扫描、配方查询）即时返回；**世界动作类工具**会被发往服务端，排进伙伴身体上的队列，由逐 tick 的任务分发器执行。
3. 执行结果回流进对话，模型据此看到发生了什么、再决定下一步。它能在一次请求里串联许多动作——你也可以随时点**停止**。
4. 历史记录在接近上下文窗口时会**自动压缩**（摘要化），所以长对话也能持续工作。

### 关于伙伴本身

- **像玩家一样持久化**——状态保存在原版 `playerdata/*.dat`（坐标、物品栏、生命、主人），登录后伙伴会回来。
- **跨维度认主**——归属按 UUID 存储与校验（原版 `getOwner()` 受维度限制，跨维度会失效，故 Numen 避开它）。
- **可恢复的死亡**——原版死亡流程照常进行（掉落、keepInventory、墓碑类模组都正常）；智能体暂停，身体在短暂延迟后于主人处重生。
- **在 Tab 列表里以玩家身份出现**，并使用自己的皮肤渲染。

## 安装与使用

### 前置与运行环境

- **需要安装在客户端与服务端两侧**（单人与服务器均可）。
- Fabric 平台需要 **Fabric API** 作为前置。
- **需要联网**，并**自备一个 LLM 服务商的 API Key**（token 由你的服务商按用量计费）。

### 上手步骤

1. 安装模组（Fabric 平台另需 Fabric API），启动一次游戏以生成配置。
2. **填入 API Key。** 按 **`G`** → **设置**（或输入 `/numen settings`），选择服务商并粘贴你的 Key。
3. **召唤伙伴：** `/numen player summon <名称>`
4. **按 `G`** 打开聊天，告诉它要做什么。

### 界面

按 **`G`** 打开伙伴（只有一个时直接进入聊天，多个时显示名册）。面板分三个标签页：

- **聊天**：可滚动的对话记录 + 一个**实时计划面板**（模型当前的待办清单）。工具运行时会显示加载动画，完成后折叠成简洁摘要，点击可展开。
- **物品**：一张只读的"角色面板"，风格仿照原版物品栏：实时随鼠标转动的 3D 立绘、盔甲与副手、2×2 合成格、生命/饥饿状态，以及完整背包。
- **设置**：服务商、API Key（默认遮蔽，可切换显示）、模型、Base URL。

屏幕左缘有一个 **HUD**，为每个伙伴显示一个小头像：闲置时缩成一条细细的金色条；当某个伙伴说话时，它的头像会连同气泡一起滑出。

### 指令

| 指令 | 说明 |
|---|---|
| `/numen player summon <名称>` | 召唤（或唤醒）一个伙伴 |
| `/numen player despawn <名称>` | 永久遣散一个伙伴 |
| `/numen settings` | 打开设置界面 |
| `/numen reset` | 重置客户端侧的智能体循环 |

## 配置

游戏内即可设置全部选项（**`G`** → 设置），也可直接编辑配置文件。

| 字段 | 说明 |
|---|---|
| `provider` | `openai`（默认）、`deepseek`、`moonshot`/`kimi`、`minimax`、`volcengine`/`doubao`、`dashscope`/`qwen` |
| `apiKey` | 你的 Key——本地存储，按游戏目录隔离 |
| `model` | 模型 id |
| `baseUrl` | 任意 OpenAI 兼容接口的可选覆盖地址 |
| `systemPrompt` | 附加在智能体提示词之前的基础人设 |

配置文件位置：**`config/numen.json`**（Fabric）或 **`config/numen-common.toml`**（NeoForge）。

## 支持的版本与加载器

- 一套代码，多平台发布：**Fabric**、**Forge**、**NeoForge**。
- 覆盖从 **1.20.1** 到最新版本的多个 Minecraft 版本（各版本对应的加载器组合以发布页为准；通常较低版本为 Forge + Fabric，较高版本为 NeoForge + Fabric）。
- 请在下载页选择与你的游戏版本、加载器相匹配的文件。

---

## 注意事项

- 你需要**自备 LLM API Key**；token 由你的服务商计费，每位玩家为自己的伙伴付费。
- **所有游戏逻辑都在本地运行**——只有对话内容会发送到你的 LLM 服务商。
- 本模组通过自然语言与模型协作，行为由模型决定，因此同一指令的执行细节可能因模型与情境而略有不同。
